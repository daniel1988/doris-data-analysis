# 用户属性分析 - 功能概述与核心逻辑 (old_frontend)

本文档整理了 `old_frontend` 中用户属性分析模块的功能定义、核心逻辑及实现方式。

---

## 1. 功能定义

用户属性分析用于针对用户维度的属性进行统计分析，评估不同用户群体的特征分布。

### 核心要素
-   **分析指标 (Metric)**：选择一个用户属性字段（如：注册时间、等级、金币数）并指定聚合方式。
-   **特殊指标**：支持“用户数”统计（对应占位符 `__TOTAL_USERS__`）。
-   **全局过滤 (Global Filters)**：针对用户属性进行的筛选。
-   **分组分析 (Grouping)**：
    -   **普通分组**：按用户属性值、区间或日期进行分组。
    -   **人群分群**：按自定义的用户分群条件进行对比分析。

---

## 2. 核心逻辑

### 2.1 指标定义
-   **属性字段**：必须属于 `user_data` (ups) 表。
-   **聚合公式**：
    -   数值型：支持 Sum, Avg, Max, Min, CountDistinct。
    -   字符型/日期型：通常仅支持 CountDistinct。
    -   **总用户数**：固定使用 `COUNT_DISTINCT(u_openid)`。

### 2.2 分组模式 (Group Type)
系统支持两种互斥的分组模式：
1.  **普通分组 (Type 1)**：传统的维度拆解。
2.  **人群分群 (Type 2)**：用户可以定义多个“分群规则”，每个分群规则包含一组过滤器，系统会统计符合各分群条件的人群指标。

### 2.3 数据处理
-   前端使用 `__TOTAL_USERS__` 作为总用户数的 UI 占位符。
-   在发送请求前，系统会将该占位符映射为真实的数据库列 `u_openid`。

## 3. 核心文件结构

用户属性分析的代码位于 `old_frontend/views/property-analysis/`：

-   [PropertyAnalysis.vue](file:///d:/gitee/dmp_admin_v2/old_frontend/views/property-analysis/PropertyAnalysis.vue)：主入口页面。
-   `components/`：UI 布局与核心卡片（指标卡片、结果视图）。
-   `composables/`：表单状态、执行引擎、报表持久化。

---

## 4. 关键特性实现

-   **属性类型感知**：在 [UserPropertyMetricCard.vue](file:///d:/gitee/dmp_admin_v2/old_frontend/views/property-analysis/components/UserPropertyMetricCard.vue) 中，根据选择的属性类型动态切换可选的聚合公式。
-   **结果可视化**：支持表格、柱状图和饼图展示用户特征分布。
-   **分群对比**：通过 `PropertyUserGroupsPanel` 支持复杂的人群特征对比分析。
