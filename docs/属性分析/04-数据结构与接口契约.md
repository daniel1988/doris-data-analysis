# 用户属性分析 - 数据结构与接口契约 (old_frontend)

本文档整理了 `old_frontend` 中用户属性分析模块的数据结构及其 API 交互细节。

---

## 1. 核心数据结构

### 1.1 `UserPropertyForm` (前端表单结构)
```typescript
interface UserPropertyForm {
  project_alias: string;
  page_size: number;
  page_num: number;
  time_zone: string;
  
  // 模式切换: 1-普通分组, 2-人群分群
  user_property_group_type: 1 | 2;

  // 分析指标
  user_property_metric: {
    name: string; // 显示名称
    metric: {
      column: { table: string; field: string }; // table 通常为 'ups'
      formula: number; // 聚合公式 (Sum, Avg, Max, Min, CountDistinct)
    };
  };

  // 过滤器
  filter_groups: {
    global_filters: {
      scope: number;
      filters: any[];
    };
    user_group_filters: any[];
    tag_filters: any[];
  };

  // 分组 (仅当 type=1 时有效)
  groups: any[];

  // 人群分群 (仅当 type=2 时有效)
  user_groups: Array<{
    alias: string;
    filter_group: any;
  }>;
}
```

## 2. API 接口契约

### 2.1 执行属性分析
-   **URL**: `/api/v1/user-property-analysis/query` (示例)
-   **Method**: `POST`
-   **Request Payload**: 包含规范化后的 `metric` 以及基于模式转换后的 `groups`/`user_groups`。
-   **Response Structure**:
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "rows": [
      {
        "group_value": "北京", // 或分群别名
        "metric_value": 12345,
        "ratio": 0.25 // 占比（部分接口支持后端返回）
      }
    ],
    "columns": ["分组维度", "指标值"],
    "count": 100,
    "sql": "SELECT ..."
  }
}
```

---

## 3. 关键映射逻辑

-   **占位符处理**：如果 `field === '__TOTAL_USERS__'`，请求时必须转换为 `u_openid` 并使用 `COUNT_DISTINCT` 公式。
-   **数据表映射**：用户属性分析的底层查询主要针对 `user_data` 表（别名为 `ups`）。
-   **结果转换**：前端 `ResultView` 会根据后端返回的 `rows` 自动计算百分比，并适配 ECharts 的饼图/柱状图。
