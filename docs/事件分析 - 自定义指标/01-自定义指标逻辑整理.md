# 事件分析 - 自定义指标逻辑整理 (old_frontend)

本文档整理了 `old_frontend` 中事件分析模块的 **自定义指标 (Custom Metrics)** 的实现逻辑、核心组件及数据交互流程。

---

## 1. 功能概述

自定义指标允许用户通过可视化界面编辑复杂的计算公式（如 `(m0 + m1) / 100`），将多个基础事件指标组合成一个新的分析维度。

### 核心特性
- **Token 化编辑**：类 IDE 的 Token 编辑体验（事件、数字、运算符）。
- **指标级过滤**：公式中的每个基础指标（Token）都可以单独配置过滤条件。
- **自动别名管理**：系统自动为公式中的基础指标分配别名（如 `m0`, `m1`），确保后端能准确解析。
- **数值格式化**：支持对计算结果进行格式化（原始、整数、两位小数、百分比）。

## 2. 核心组件 (Core Components)

### 2.1 [CustomFormulaInput.vue](file:///d:/gitee/dmp_admin_v2/old_frontend/components/common/CustomFormulaInput.vue)
- **作用**：公式编辑的可视化输入框。
- **功能**：
    - 管理 Token 序列。
    - 弹出 `FormulaPanel` 进行 Token 选择。
    - 弹出 `MetricPropertyPicker` 进行属性与聚合方式选择。
    - 弹出 `BaseFilterItem` 进行指标级过滤配置。

### 2.2 [CustomFormulaPanel.vue](file:///d:/gitee/dmp_admin_v2/old_frontend/views/analysis/components/metrics/CustomFormulaPanel.vue)
- **作用**：作为 `MetricItem` 切换到自定义模式时的容器。
- **功能**：将 `form` 中的指标数据传递给 `CustomFormulaInput` 并监听更新。

### 2.3 [MetricItem.vue](file:///d:/gitee/dmp_admin_v2/old_frontend/views/analysis/components/metrics/MetricItem.vue)
- **作用**：单个指标的容器，支持普通模式与自定义模式切换。
- **更新**：已增加普通指标的 **数值格式化选择器**，保持 UI 体验一致性。

## 3. 组合式逻辑 (Composables)

### 3.1 [useCustomFormula.ts](file:///d:/gitee/dmp_admin_v2/old_frontend/views/analysis/composables/useCustomFormula.ts)
- **核心职能**：双向同步 `custom_web_metric` (前端 UI 状态) 与 `custom_metric` (API Payload)。
- **同步逻辑**：
    - 遍历 `eventMetrics` Token。
    - 提取事件指标、数字、运算符。
    - 自动生成 `custom_formula` 字符串（如 `(指标A + 指标B)`）。
    - 规范化特殊字段（如 `__TOTAL_TIMES__` -> `e_openid`）。

### 3.2 [useMetricEditing.ts](file:///d:/gitee/dmp_admin_v2/old_frontend/views/analysis/composables/useMetricEditing.ts)
- **核心职能**：处理指标的“普通 <-> 自定义”模式转换。
- **逻辑细节**：
    - **普通 -> 自定义**：将当前指标作为公式的第一个 Token（`m0`），并初始化自定义指标结构。
    - **自定义 -> 普通**：取公式中的第一个事件指标作为新的普通指标，并恢复原有结构。

## 4. 数据结构 (Data Structure)

### 4.1 前端展示结构 (`custom_web_metric`)
```json
{
  "eventMetrics": [
    { "type": "operator", "value": "(" },
    { "type": "event", "value": { "e_event_id": "login", "metric": { ... }, "filter_group": { ... } } },
    { "type": "operator", "value": "+" },
    { "type": "number", "value": "100" },
    { "type": "operator", "value": ")" }
  ]
}
```

### 4.2 API 请求结构 (`custom_metric`)
```json
{
  "event_metrics": [
    { "e_event_id": "login", "name": "自定义指标1_0", "metric": { ... }, "filter_group": { ... } }
  ],
  "custom_formula": "(自定义指标1_0 + 100)",
  "format": "decimal"
}
```

## 5. 开发建议与注意事项

1. **Token 唯一性**：每个 Token 必须有 `_uid` 以保证 Vue `v-for` 渲染性能及状态正确。
2. **事件 ID 联动**：在自定义公式内切换事件 ID 时，需通过 `eventsKey` 强制刷新相关子组件（如属性选择器）。
3. **字段映射**：前端展示的 `__TOTAL_TIMES__` 在构建 Payload 时必须映射为 `e_openid` + `COUNT` 公式。
