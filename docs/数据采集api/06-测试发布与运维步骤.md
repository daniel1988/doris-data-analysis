# 06-测试发布与运维步骤

## 1. 测试计划

### 1.1 单元测试
- DTO 绑定与校验测试
- 标准化函数测试
- 操作符与边界值测试（空 events、超长字段、非法时间）

### 1.2 集成测试
- 单事件/单用户批量/多用户批量三条链路
- 成功、部分失败、全部失败场景
- 队列不可用、DB 不可用场景
- 签名校验场景：
  - Header 缺失（`X-Project-Alias/X-Timestamp/X-Nonce/X-Sign`）
  - 错误签名（secret 不匹配）
  - 时间戳超窗（超过 ±5 分钟）
  - nonce 重放（10 分钟内重复）

### 1.3 压测
- 目标 QPS（按业务评估）
- 核心指标：
  - p95/p99 延迟
  - 错误率
  - accepted/dropped 比例

## 2. 发布步骤

1. 先在测试环境联调 SDK 与服务端上报
2. 灰度开启（按 project_alias 白名单）
3. 观察监控与错误日志
4. 全量放开并保留回滚开关

## 3. 监控与告警

建议最少监控项：
- 请求总量与成功率
- 各错误码数量
- 下游写入失败率
- 队列堆积长度
- 单项目限流触发次数
- 签名校验失败率（40102）
- nonce 重放拦截次数（40103）
- 时间戳超窗次数（40104）

## 4. 运维排障手册要点

- 按 `trace_id` 串联 API 日志与下游消费日志
- 可按 `project_alias + event_id + 时间范围` 回放失败数据
- 发生高错误率时可快速降级为“仅接收不下发”模式

## 5. 本步骤验收标准

- 有完整压测报告与灰度方案
- 有可执行的回滚预案
- 关键告警均可触发并通知
