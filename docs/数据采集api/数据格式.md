


## 单用户多事件
单用户多事件可用于 sdk 客户端批量事件上报， 主要用途是降低客户端的请求。
```
{
  "openid": "11002",
  "channel_id": "10401",
  "device_id": "4a58162c-3b92-4f7a-b987-4bfddb904ac2",
  "events": [
    {
      "event_id": "sys.ad",
      "event_time": 1657092829,
      "event_name": "广告",
      "properties": {
        "order_id": "no12312412",
        "ecpm": 6.8
      }
    },
    {
      "event_id": "sys.activate",
      "event_name": "激活",
      "event_time": 1657092829,
      "properties": {
      }
    }
  ]
}
```

## 单事件
支持单事件上报。 为减少网络开销，数据上报时， 可尽量用 单用户事件 及 多用户事件 结构进行上报。 
```
{
  "openid": "11002",
  "channel_id": "10401",
  "device_id": "4a58162c-3b92-4f7a-b987-4bfddb904ac2",
  "event_id": "sys.ad",
  "event_time": 1657092829,
  "event_name": "广告",
  "properties": {
    "order_id": "no12312412",
    "ecpm": 6.8
  }
}
```

## properties 字段入库规则

`properties` 中的自定义字段在写入 `event_data` 表时，需要先做字段名解析并统一追加 `e_` 前缀：

- 上报：`"properties": { "order_id": "no123", "ecpm": 6.8 }`
- 入库列：`e_order_id`, `e_ecpm`

规则要求：

1. `properties` 的 key 在进入 Doris 前统一转为 `e_{key}` 列名
2. 如果 key 本身已带 `e_` 前缀，则不重复追加
3. 空 key 或非法字符 key 不入库，并记录字段告警
4. 解析后字段需与 `event_data` 表字段校验流程联动（不存在字段按策略处理）
