# 01-开发步骤总览

## 1. 目标

基于 [数据格式.md](file:///d:/gitee/dmp_admin_v2/docs/数据采集api/数据格式.md) 中定义的三种上报形态，落地一套 Golang 数据采集 API：
- 单事件上报
- 单用户多事件上报
- 多用户多事件上报

本总览用于定义整体实施顺序与交付边界。

## 2. 实施顺序

1. 请求模型与接口契约落地（DTO + 校验规则）
2. API 层开发（路由、参数绑定、统一响应）
3. Service 层开发（标准化、批量拆解、基础校验）
4. 存储链路开发（先按分钟落本地日志，再按分钟解析并通过 Stream Load 写 Doris）
5. 可观测与发布（日志、指标、压测、灰度）

## 3. 分阶段交付

### 阶段 A（MVP）
- 完成 3 个采集接口
- 完成统一响应结构
- 完成基础字段校验
- 完成签名校验（HMAC-SHA256 + nonce 防重放）
- 事件按分钟写本地日志文件，确保可追踪与可回放

### 阶段 B（生产可用）
- 完成分钟日志解析任务并稳定通过 Stream Load 导入 Doris
- 增加签名校验、限流、幂等
- 增加失败重试与死信处理

### 阶段 C（可观测增强）
- 增加请求链路追踪
- 增加采集成功率、延迟、错误码监控
- 增加按项目维度的数据质量看板
