# 03-后端接口与服务实现

后端需要提供看板及其组件的增删改查接口，并集成已有的 `DashboardService`。

## 1. 接口设计 (API)

### 1.1 看板管理
- `GET /api/v1/dashboard/list`: 获取当前项目下的看板列表
- `POST /api/v1/dashboard`: 创建新看板
- `PUT /api/v1/dashboard`: 更新看板基本信息
- `DELETE /api/v1/dashboard/:id`: 删除看板

### 1.2 看板项管理
- `POST /api/v1/dashboard/item`: 向看板添加报表项
- `PUT /api/v1/dashboard/item`: 更新项的布局或配置覆盖
- `DELETE /api/v1/dashboard/item/:id`: 从看板移除项
- `PUT /api/v1/dashboard/items/batch`: 批量更新看板项（如保存布局时使用）

## 2. 后端逻辑实现

- **路由注册**：在 `internal/api/router.go` 中补全看板 API 组。
- **Handler 实现**：新建 `DashboardHandler` 处理请求参数校验及调用 Service 层。
- **Service 调用**：复用并完善 `internal/service/analytics/dashboard_service.go` 中的逻辑。
- **Doris 语法兼容性优化**：
  - **查询优化**：避免使用 `.First()`、`.Take()`，统一使用切片配合 `.Find()` 模式，防止自动生成 `LIMIT 1` 子句。
  - **更新/删除优化**：明确指定 `Where("id = ?", id)`，并确保生成的 SQL 不包含 `LIMIT`。
- **级联删除**：在删除看板时，务必开启事务，同步清理 `dashboard_items` 表中关联的记录，防止产生脏数据。
