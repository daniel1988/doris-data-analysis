# 08-报表展示优化专项方案

## 1. 核心目标
优化数据看板内报表的默认展示体验。通过“默认表格、按需切换图表”的策略，提升看板的首屏加载性能及多报表下的数据可读性。

## 2. 报表展示模型规范

| 分析模型 | 默认展示形式 | 是否支持切换图表 | 图表类型说明 |
| :--- | :--- | :--- | :--- |
| **事件分析 (Events)** | 数据表 (Data Table) | 是 | 折线图、柱状图、饼图等 |
| **漏斗分析 (Funnel)** | 数据表 (Data Table) | 是 | 漏斗图、柱状图等 |
| **属性分析 (Property)** | 数据表 (Data Table) | 是 | 柱状图、饼图等 |
| **分布分析 (Scatter)** | 数据表 (Data Table) | 是 | 柱状图等 |
| **留存分析 (Retention)** | 数据表 (Matrix) | **否** | 仅支持留存矩阵表格展示 |
| **汇总指标卡 (KPI/Stat)** | 大数字汇总 (Stat Card) | **否** | 展示核心 KPI 及环比趋势 |

## 3. 核心功能逻辑

### 3.1 默认展示策略 (Default Display)
- 所有分析模型（除留存分析和汇总指标卡外）在看板加载时，**统一默认展示为数据表格**。
- 表格需支持：
  - 前端分页与排序。
  - 列宽自适应。
  - 基于 `rows` 与 `columns` 数据的标准渲染。

### 3.2 图表切换机制 (Chart Toggling)
- **切换入口**：在 `DashboardItemCard` 的 Header 操作区域增加“切换图表/表格”按钮。
- **图表渲染**：
  - 切换为图表后，复用分析页面的 `ChartView` 组件。
  - 图表类型（折线、柱状等）根据报表保存时的 `vizMode` 自动识别。
- **状态保持**：
  - 切换状态仅为前端运行时状态（Session Only）。
  - **页面刷新后，报表将恢复为默认的“数据表格”展示**。

### 3.3 布局尺寸快捷切换 (Size Toggling)
- **切换入口**：在看板项卡片右上角的“更多操作（三点图标）”下拉菜单中增加切换选项。
- **切换逻辑**：
  - **设为中图**：将该组件在栅格系统中的宽度调整为 `12`（占看板宽度的 50%）。
  - **设为大图**：将该组件在栅格系统中的宽度调整为 `24`（占看板宽度的 100%）。
- **实时生效**：点击切换后立即更新 `vue-grid-layout` 的布局数据，并调用后端批量保存接口持久化该布局变更。

### 3.4 留存分析特殊性
- 留存分析由于其数据结构的复杂性（矩阵结构），在看板中**固定展示为数据表格**。
- 移除卡片 Header 中的切换按钮。

## 4. 关键组件与实现路径

### 4.1 涉及组件
- **DashboardItemCard.vue**：控制卡片 Header 的切换按钮显示，并根据 `showTable` 状态分发渲染。
- **TableView.vue**：通用的表格展示组件。
- **ChartView.vue**：通用的 ECharts 图表渲染组件。

### 4.2 状态控制 (伪代码)
```typescript
// DashboardItemCard.vue
const showTable = ref(true) // 默认开启表格模式

// 留存分析强制为表格
const canToggle = computed(() => report.value?.category !== 'retention')

const toggleView = () => {
  if (canToggle.value) {
    showTable.value = !showTable.value
  }
}
```

## 5. 验收口径
1. **默认展示**：进入看板，所有支持表格的报表（事件、漏斗等）首屏均展示为表格。
2. **切换功能**：点击切换按钮，图表能够正确渲染，且样式与分析页一致。
3. **留存一致性**：留存分析报表始终以表格形式展示，且没有切换按钮。
4. **刷新重置**：手动刷新浏览器页面，所有已切换为图表的报表应重置为表格展示。
