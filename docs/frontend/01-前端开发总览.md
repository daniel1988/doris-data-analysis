# 01-前端开发总览与第一阶段指南

本文档旨在为 `dmp_admin_v2` 前端（Vue3 + TS）的开发提供全局指导和分步实施计划。

## 1. 基础环境搭建 (Step 0)

在开始业务模块开发前，需要完成以下基础建设：

1. **项目初始化**：在 `frontend` 目录下初始化 Vite + Vue3 + TS + Element Plus。
2. **全局布局 (Layout)**：实现侧边栏、顶部导航、面包屑及多页签 (Tabs) 逻辑。
3. **网络请求封装 (Axios)**：
   - 基础 URL 配置
   - 请求/响应拦截器（处理 Token、全局错误提示）
4. **状态管理 (Pinia)**：建立 `user` (用户信息/权限) 和 `app` (配置/页签) Store。
5. **路由规划**：根据模块划分建立动态路由配置。

## 2. 模块导航与结构

系统采用 **顶部导航 + 侧边二级导航** 的混合布局模式：

- **顶部导航**: 放置核心功能模块。
  - **看板管理 (Dashboard)**: 分析结果的聚合展示。
  - **分析模块 (Analytics)**: 提供多种分析模型。
  - **元数据管理 (Metadata)**: 数据资产的定义与配置。
  - **系统管理 (System)**: 权限、用户与租户管理。

- **二级导航 (Sidebar)**: 各模块内部的功能切换。
  - **看板管理**: 左侧展示生成的看板列表。
  - **分析模块**: 包含 `事件分析`、`留存分析`、`漏斗分析` 等。
  - **元数据管理**: 包含 `元字段`、`元事件`、`维度` 等。

## 3. 用户体验与视觉规范 (UX/UI Guidelines)

为了保证系统的一致性与专业感，开发时请遵循以下规范：

### 3.1 视觉风格 (Visual Style)
- **色彩体系**: 
  - 主色：使用科技蓝（如 `#409EFF`）作为品牌色。
  - 辅助色：成功（绿）、警告（黄）、危险（红）需严格遵循 Element Plus 预设。
  - 背景：主内容区背景色建议为浅灰（`#f0f2f5`），卡片背景为纯白。
- **字体与间距**:
  - 标题、正文、辅助文字需有明显的字号和颜色深度区分。
  - 统一使用 `16px` 或 `24px` 作为标准的卡片间距。

### 3.2 交互体验 (User Experience)
- **反馈机制**:
  - 所有按钮点击、数据提交必须有 Loading 状态或禁用状态。
  - 操作成功/失败需有明确的 `Message` 提示。
- **空状态处理**:
  - 图表无数据、列表无结果时，必须展示 `el-empty` 或自定义占位图，避免页面留白。
- **加载优化**:
  - 复杂图表和长列表建议使用骨架屏 (`el-skeleton`) 进行占位，提升感知速度。
- **响应式适配**:
  - 核心分析页面需兼容不同分辨率，确保在 1366px 到 4K 屏幕下布局不乱。

## 4. 页面开发通用步骤

针对每个业务页面，遵循以下标准流程：

1. **路由与目录创建**：在 `src/views` 下建立对应文件，并在 `router` 中注册。
2. **UI 布局设计**：使用 Element Plus 容器组件完成页面框架。
3. **状态/变量定义**：定义查询参数、数据列表、加载状态等响应式数据。
4. **API 接口定义**：在 `src/api` 下创建对应模块的请求函数。
5. **逻辑实现**：
   - 数据获取与展示。
   - 图表集成（ECharts）。
   - 交互逻辑（筛选、分页、详情跳转）。
6. **优化与自测**：处理加载态、空状态、异常提示及响应式适配。
