# 04-分析模块开发文档

行为分析模块是 `dmp_admin_v2` 的核心价值，包含事件分析、漏斗分析、留存分析等能力。

## 1. 模块概览

- **页面结构**: 采用 **顶部导航 (下拉子菜单) + 左侧配置面板** 布局。
- **主要页面**: 
  - **事件分析 (`EventAnalysis.vue`)**: 基础行为趋势分析。
  - **留存分析 (`RetentionAnalysis.vue`)**: 用户粘性与回访分析。
  - **漏斗分析 (`FunnelAnalysis.vue`)**: 转化路径与流失分析。
- **功能切换**: 通过顶部导航“分析”的下拉菜单进行页面切换。
- **分析配置面板 (Config-Panel)**: 固定在页面左侧，承载具体的分析配置项（事件、过滤、分组），方便用户从左往右进行操作。

## 2. 核心功能设计

### 2.1 事件分析 (Event Analysis)
- **输入**: 选择事件、属性过滤、时间跨度、分群粒度。
- **输出**: 趋势图（折线/面积）、数据表格（聚合结果）。
- **步骤**:
  1. 建立事件选择器（基于元数据）。
  2. 实现动态属性过滤列表。
  3. 配置 ECharts 趋势图，支持对比不同维度。

### 2.2 漏斗分析 (Funnel Analysis)
- **输入**: 定义步骤（事件 A -> 事件 B -> 事件 C）、时间限制、窗口期。
- **输出**: 漏斗图、各步转化率、中位耗时。
- **步骤**:
  1. 建立可拖拽排序的步骤配置器。
  2. 配置漏斗图展示转化流失。
  3. 支持点击流失环节查看明细或进行分群。

### 2.3 留存分析 (Retention Analysis)
- **输入**: 起始事件、回访事件、时间窗口（日/周/月）。
- **输出**: 留存热图、留存曲线。
- **步骤**:
  1. 设计留存矩阵表格。
  2. 使用热图色彩展示不同留存率阶梯。

## 3. 开发步骤建议

### 第一阶段：通用分析组件开发
1. **分析配置面板 (`AnalysisConfigPanel.vue`)**: 固定在页面**右侧**，包含事件选择、属性筛选（且/或逻辑）、全局过滤。
2. **时间选择器 (`DateRangePicker.vue`)**: 支持常用的 7天、30天、昨天、今天及自定义范围，通常放在顶部操作栏。

### 第二阶段：事件分析页面
1. 建立基于 `TopNavLayout` 的页面布局：
   - **顶部**: 导航菜单、多页签。
   - **左侧/中间**: 图表展示区、原始数据预览。
   - **右侧**: 展开式的 `AnalysisConfigPanel`。
2. 实现 API 对接，获取 Doris 聚合后的趋势数据。
3. 集成 ECharts 展示数据，支持多维度切换。

### 第三阶段：复杂分析页面（漏斗/留存）
1. 实现步骤编辑器（漏斗步骤添加/删除/排序）。
2. 针对漏斗和留存的特殊数据格式进行处理。
3. 增加分析结果的保存功能（保存为看板图表）。

## 4. 数据契约示例

### 4.1 分析请求 (POST /api/v1/analytics/query)
```json
{
  "analysis_type": "event",
  "events": ["login", "view_page"],
  "filters": [
    {"property": "os", "op": "==", "value": "ios"}
  ],
  "date_range": ["2026-03-01", "2026-03-13"],
  "granularity": "day"
}
```

### 4.2 响应结果
```json
{
  "series": [
    {"name": "login", "data": [100, 120, 150]}
  ],
  "categories": ["2026-03-01", "2026-03-02", "2026-03-03"]
}
```
