# 02-用户登录模块开发文档

用户登录是系统的入口，负责身份验证、Token 获取及基础权限初始化。

## 1. 模块概览 (当前阶段：使用 Mock 数据)

- **页面**: `Login.vue`
- **核心逻辑**:
  - 表单校验（账号、密码）。
  - **Mock 机制**: 现阶段通过前端硬编码逻辑校验（账号 `admin`，密码 `123456`），模拟 Token 返回并存储。
  - 存储 Token 到本地 (localStorage/Cookie) 及 Pinia Store。
  - 登录成功后跳转至首页，并触发菜单和权限的异步获取。

## 2. 界面设计建议

- **风格**: 简洁、专业，支持响应式适配。
- **元素**:
  - 背景：可使用简单的几何图形或行为分析相关的背景图。
  - 表单：居中展示，包含 Logo、标题、账号输入框、密码输入框（支持明文切换）、验证码（可选）、记住我。

## 3. 开发步骤

### 第一阶段：UI 实现
1. 在 `src/views/login/Login.vue` 创建基础页面。
2. 使用 Element Plus 的 `el-form` 和 `el-input` 构建表单。
3. 添加基础的 CSS 样式。

### 第二阶段：逻辑处理
1. **API 对接**: 在 `src/api/auth.ts` 中定义 `login` 接口。
2. **状态管理**: 在 `src/store/user.ts` 中实现 `login` action，处理 Token 存储。
3. **路由守卫**: 在 `src/router/guard.ts` 中实现全局守卫，拦截未登录访问。

### 第三阶段：体验优化
1. **加载态**: 登录按钮点击后的 Loading 状态。
2. **异常处理**: 处理账号密码错误、网络超时等提示。
3. **回车登录**: 支持在输入框按回车键触发登录。

## 4. 数据契约示例

### 4.1 登录请求 (POST /api/v1/auth/login)
```json
{
  "username": "admin",
  "password": "hashed_password",
  "captcha": "1234"
}
```

### 4.2 响应结果
```json
{
  "access_token": "ey...",
  "refresh_token": "ey...",
  "expires_in": 3600,
  "user_info": {
    "id": "u_001",
    "name": "管理员",
    "avatar": "..."
  }
}
```
