# 06-元数据管理开发文档

元数据管理模块负责定义事件和属性，为行为分析提供语义支持。

## 1. 模块概览

- **页面结构**: 采用 **顶部导航 (下拉子菜单)** 布局。
- **主要页面**: 
  - **元事件 (`EventList.vue`)**: 定义与管理上报的行为事件。
  - **元字段 (`FieldList.vue`)**: 定义与管理属性字段。
  - **维度 (`DimensionList.vue`)**: 管理维度表与关联关系。
- **功能切换**: 通过顶部导航“元数据”的下拉菜单进行功能切换。

## 2. 核心功能设计

### 2.1 事件列表 (Event List)
- **展示内容**: 事件名、显示名、所属分类、状态（显示/隐藏）、描述。
- **管理功能**: 编辑显示名、启用/停用事件、配置入库策略。

### 2.2 属性列表 (Property List)
- **分类**: 
  - 预置属性（OS、浏览器、IP、城市等）。
  - 自定义属性（订单号、商品 ID、价格等）。
- **管理功能**: 修改显示名、数据类型校验、值范围定义（枚举值）。

### 2.3 埋点规范 (Tracking Specification)
- **展示方式**: 以文档或表格形式展示各个版本的埋点需求与状态。
- **交互**: 导出埋点协议（JSON/Excel）供开发集成。

## 3. 开发步骤建议

### 第一阶段：元数据展示
1. **事件列表页 (`EventList.vue`)**: 展示 Doris 中已上报的事件清单。
2. **属性详情 (`PropertyDetail.vue`)**: 点击事件查看其关联的所有属性及其类型。

### 第二阶段：元数据配置
1. **编辑表单**: 实现修改显示名、添加备注、设置分类等功能。
2. **分类管理**: 建立事件分类树，支持按业务线或功能模块归类。

### 第三阶段：虚拟属性与高级功能
1. **计算属性**: 实现简单的属性加减乘除（如：订单总额 = 单价 * 数量）。
2. **属性字典**: 维护枚举值对应的可读标签（如：0 -> 女, 1 -> 男）。

## 4. 数据契约示例

### 4.1 事件元数据 (GET /api/v1/metadata/events)
```json
[
  {
    "name": "buy_product",
    "display_name": "购买商品",
    "category": "交易",
    "is_visible": true,
    "property_count": 12,
    "last_active": "2026-03-13T09:45:00Z"
  }
]
```

### 4.2 属性定义 (GET /api/v1/metadata/properties?event=buy_product)
```json
[
  {
    "name": "price",
    "display_name": "单价",
    "type": "number",
    "unit": "CNY",
    "is_virtual": false
  }
]
```
