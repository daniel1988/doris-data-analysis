# 03-后台页面布局开发文档 (2026-03-13 修订版 v3)

`dmp_admin_v2` 采用 **顶部居中导航 + 模块化侧边栏** 的现代化布局方案。

## 1. 布局方案结构

- **顶部导航栏 (Header)**: 
  - **Logo**: 位于最左侧。
  - **一级模块 (居中)**: 看板、分析、元数据、系统管理。
  - 菜单规则: 强制全部显示，禁止通过省略号 (...) 隐藏。
  - **用户信息**: 位于最右侧。
- **左侧边栏 (Sidebar)**: 
  - **分析模块**: 承载具体的 **分析配置面板 (Config-Panel)**。
  - **元数据 & 系统管理**: 承载 **二级导航菜单**，提供深层级的操作入口。
- **主体内容区 (Main Content)**: 展示业务核心内容。

## 2. 交互逻辑

- **全局跳转**: 通过顶部居中的一级导航进行大模块切换。
- **功能操作**: 
  - 分析类页面在左侧进行参数配置。
  - 管理类页面在左侧进行子功能导航。

### 2.3 动态配置面板 (Config-Panel)
- **位置**: 固定在页面右侧。
- **功能**: 针对分析模块，放置复杂的过滤条件、分组设置等。支持展开/折叠。

## 3. 开发步骤建议

### 第一阶段：统一布局框架搭建
1. 实现 `SidebarLayout.vue`，整合左侧菜单和顶部导航。
2. 移除原有的 `TopNavLayout` 和 `RightMenuLayout`。

### 第二阶段：通用组件优化
1. **Sidebar**: 实现可收缩的垂直菜单。
2. **Topbar**: 优化面包屑与多页签的展示。

### 第三阶段：路由适配
1. 更新路由配置，将所有受保护页面挂载到 `SidebarLayout` 下。
2. 确保看板页面与分析页面在统一布局下正常显示。

## 4. 样式建议

- **侧边栏宽度**: 展开时 200-240px，折叠时 64px。
- **顶部导航高度**: 50-60px。
- **配置面板宽度**: 右侧面板建议 300px。
- **色彩**: 侧边栏建议使用深色（如 `#304156`）或纯白背景，突出品牌感。
