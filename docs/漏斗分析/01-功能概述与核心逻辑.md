# 漏斗分析 - 功能概述与核心逻辑 (old_frontend)

本文档整理了 `old_frontend` 中漏斗分析模块的功能定义、核心逻辑及实现方式。

---

## 1. 功能定义

漏斗分析用于衡量用户在特定流程（如注册流程、购买流程）中，从一个步骤到下一个步骤的转化情况。

### 核心要素
-   **步骤 (Steps)**：用户需要依次完成的行为序列（最少 2 步，最多 8 步）。
-   **转化窗口期 (Window Num)**：用户从第一步到最后一步必须在多长时间内完成。
-   **时间粒度 (Time Grain)**：分析的时间单位（天、周、月）。
-   **全局过滤 (Global Filters)**：对整个漏斗流程的用户进行筛选。
-   **分组分析 (Grouping)**：按用户属性或事件属性拆解漏斗，对比不同人群的转化率。

---

## 2. 核心逻辑

### 2.1 步骤定义
漏斗中的每个步骤都是一个事件指标，通常使用 **Bitmap Union (去重计数)** 来统计用户数。
每个步骤可以拥有独立的 **步骤内过滤器**。

### 2.2 转化规则
-   **有序性**：用户必须按照定义的步骤顺序执行行为（A -> B -> C）。
-   **时间窗口限制**：从第一步发生的时间开始算起，后续步骤必须在窗口期内完成。
-   **第 0 步 (Step 0)**：作为漏斗的基准，转化率为 100%。

### 2.3 关键指标
-   **转化人数**：到达该步骤的去重用户数。
-   **转化率 (Conversion Rate)**：
    -   **总转化率**：`当前步骤人数 / 第一步人数 * 100%`。
    -   **步骤转化率**：`当前步骤人数 / 上一步骤人数 * 100%`。
-   **流失人数**：`上一步骤人数 - 当前步骤人数`。

## 3. 核心文件结构

漏斗分析的主要代码位于 `old_frontend/views/funnel/` 目录下：

-   [FunnelAnalysis.vue](file:///d:/gitee/dmp_admin_v2/old_frontend/views/funnel/FunnelAnalysis.vue)：漏斗分析主入口。
-   `components/`：包含漏斗配置、步骤列表、结果展示等 UI 组件。
-   `composables/`：封装了表单管理、接口请求、报表持久化等业务逻辑。

---

## 4. 关键特性实现

-   **步骤重排序**：支持通过拖拽或按钮调整漏斗步骤的先后顺序。
-   **动态步骤名**：当用户选择事件后，系统会自动根据事件名生成步骤标题（如“步骤 1：启动应用”）。
-   **结果可视化**：支持柱状图（漏斗图样式）和详细数据表格展示。
-   **报表保存与加载**：支持将漏斗配置保存为报表，方便后续直接查看。
