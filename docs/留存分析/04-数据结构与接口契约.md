# 留存分析 - 数据结构与接口契约 (old_frontend)

本文档整理了 `old_frontend` 中留存分析模块的数据结构及其 API 交互细节。

---

## 1. 核心数据结构

### 1.1 `RetentionForm` (前端表单结构)
```typescript
interface RetentionForm {
  project_alias: string;
  page_size: number;
  page_num: number;
  time_zone: string;
  
  // 初始事件指标
  init_event_metric: {
    e_event_id: string;
    name: string;
    type: number; // 1-Normal
    metric: {
      column: { table: string; field: string };
      formula: number; // MF.COUNT_DISTINCT (2)
      format: string;
    };
    filter_groups: any[];
  };

  // 结束事件指标
  end_event_metric: {
    e_event_id: string;
    name: string;
    type: number; // 1-Normal
    metric: {
      column: { table: string; field: string };
      formula: number; // MF.COUNT_DISTINCT (2)
      format: string;
    };
    filter_groups: any[];
  };

  // 时间粒度
  time_grain: {
    column: { table: string; field: string };
    interval: number; // 1-Hour, 2-Day, 3-Week, 4-Month
    window_num: number;
  };

  // 留存天数
  day_n_array: number[]; // 如 [1, 2, 3, 4, 5, 6, 7, 14, 30]

  // 全局过滤
  global_filter_groups: {
    global_filters: {
      scope: number; // 1-And, 0-Or
      filters: any[];
      tag_filters: any[];
      user_group_filters: any[];
    };
    query_dates: string[]; // [startDate, endDate]
  };

  // 分组
  groups: any[];
}
```

## 2. API 接口契约

### 2.1 执行留存分析
-   **URL**: `/api/v1/retention-analysis/query` (示例)
-   **Method**: `POST`
-   **Request Payload**: 同 `RetentionForm`。
-   **Response Structure**:
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "rows": [
      {
        "group_values": ["Value1", "Value2"], // 分组值（如有）
        "date": "2023-10-01", // 初始日期
        "init_user_count": 1000, // 初始用户数 (Day 0)
        "retention_data": [
          { "day": 1, "user_count": 468, "rate": 0.468 },
          { "day": 7, "user_count": 248, "rate": 0.248 }
          // ... 其他配置的天数
        ]
      }
    ],
    "columns": ["分组1", "分组2", "日期", "初始人数", "第1天留存", "第7天留存"]
  }
}
```

---

## 3. 关键字段映射

-   **Formula (MF.COUNT_DISTINCT)**：留存分析中固定使用 2（去重计数），后端通常根据此值统计独立用户。
-   **Interval**：控制结果按天、周或月进行聚合。
-   **Window Num**：控制查询的时间窗口，在留存分析中，通常与 `query_dates` 配合使用。
