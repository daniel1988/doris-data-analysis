# 留存分析 - 功能概述与核心逻辑 (old_frontend)

本文档整理了 `old_frontend` 中留存分析模块的功能定义、核心逻辑及实现方式。

---

## 1. 功能定义

留存分析用于评估用户在完成某一初始行为后，在后续特定时间段内再次完成某一结束行为的比例。

### 核心要素
-   **初始事件 (Initial Event)**：用户执行的第一步操作（如：注册、登录、启动）。
-   **结束事件 (End Event)**：用户执行的第二步操作（如：购买、再次登录、点击）。
-   **留存时间 (Retention Days)**：用户在初始事件发生后的第 N 天（或周/月）再次访问的间隔。
-   **时间粒度 (Time Grain)**：分析的时间单位，支持 天 (Day)、周 (Week)、月 (Month)。

---

## 2. 核心逻辑

### 2.1 指标定义
留存分析中，指标通常使用 **去重计数 (COUNT_DISTINCT)**，即关注的是执行了行为的“独立用户数”。

### 2.2 计算规则
-   **第 0 天 (Day 0)**：初始事件发生当日，通常作为 100% 的基准。
-   **第 N 天 (Day N) 留存**：在初始事件发生后的第 N 天，完成结束事件的用户数。
-   **留存率 (Retention Rate)**：`第 N 天留存用户数 / 初始事件用户数 * 100%`。

---

## 3. 核心文件结构

留存分析的主要代码位于 `old_frontend/views/retention/` 目录下：

-   [RetentionAnalysis.vue](file:///d:/gitee/dmp_admin_v2/old_frontend/views/retention/RetentionAnalysis.vue)：主入口页面。
-   `components/`：UI 组件，负责交互与展示。
-   `composables/`：业务逻辑，负责状态管理、表单校验、API 请求。

---

## 4. 关键特性实现

-   **自定义留存天数**：支持用户配置任意 N 天（最多 10 个）进行对比分析。
-   **全局过滤**：支持对所有参与计算的事件进行统一的属性过滤、标签过滤、分群过滤。
-   **分组分析**：支持按照用户属性或事件属性进行维度拆解，对比不同人群的留存差异。
-   **报表持久化**：支持将分析配置保存为报表，并能从报表中心重新加载。
