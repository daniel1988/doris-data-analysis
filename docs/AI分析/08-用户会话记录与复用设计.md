# AI 分析会话记录功能设计与开发规划

## 1. 背景与需求
在当前的「智能分析 (AI Chat)」模块中，用户通过自然语言生成的 SQL 和图表会在页面刷新后丢失。为了方便用户沉淀有价值的分析资产，以及快速复用历史分析结果，需要增加**用户会话记录 (Session History)** 功能。

核心需求如下：
1. **主动保存**：由用户判断 AI 返回的数据/图表是否有效，若有效，用户可以主动点击“保存”或“点赞/确认”按钮将其沉淀到数据库中。
2. **记录信息**：保存时需要记录用户的提问 (Query)、AI 生成的 SQL、图表类型 (Viz Type)、坐标轴配置以及回答文本 (Narrative) 等关键信息。
3. **历史查看与复用**：在 AI 分析界面的头部区域增加一个按钮，点击后侧边滑出抽屉 (Drawer) 展示历史保存的会话记录。用户点击某条记录即可直接将数据和图表加载到当前分析面板中，无需再次消耗 AI Token。

## 2. 数据模型设计

需要在 `dmp_center` 数据库中新增一张会话记录表 `ai_chat_sessions`：

```sql
CREATE TABLE IF NOT EXISTS dmp_center.ai_chat_sessions (
  `id`              BIGINT NOT NULL AUTO_INCREMENT COMMENT '主键ID',
  `project_alias`   VARCHAR(58) NOT NULL COMMENT '项目别名',
  `user_id`         BIGINT NOT NULL COMMENT '保存该记录的用户ID',
  `user_query`      TEXT NOT NULL COMMENT '用户的自然语言提问',
  `llm_sql`         TEXT NOT NULL COMMENT 'AI 生成并验证通过的 SQL',
  `viz_type`        VARCHAR(32) NULL COMMENT '图表类型 (bar, line, pie, table 等)',
  `x_axis`          VARCHAR(128) NULL COMMENT 'X轴字段',
  `y_axis`          VARCHAR(128) NULL COMMENT 'Y轴字段',
  `narrative`       TEXT NULL COMMENT 'AI 给出的结论描述',
  `create_time`     DATETIME DEFAULT CURRENT_TIMESTAMP,
  `update_time`     DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
) ENGINE=OLAP
UNIQUE KEY(`id`)
DISTRIBUTED BY HASH(`id`) BUCKETS 1
PROPERTIES (
  "enable_unique_key_merge_on_write" = "true"
);
```

## 3. 后端接口设计 (API)

在 `ai_chat_handler` 相关的路由组中增加以下 RESTful 接口：

| 接口路径 | 方法 | 说明 | 权限 |
| --- | --- | --- | --- |
| `/api/v1/analytics/ai/sessions` | `POST` | 保存一条有效的 AI 会话记录 | 当前登录用户 |
| `/api/v1/analytics/ai/sessions` | `GET` | 获取当前项目、当前用户的历史会话列表 | 当前登录用户 |
| `/api/v1/analytics/ai/sessions/:id` | `DELETE` | 删除某条历史会话 | 当前登录用户 |
| `/api/v1/analytics/ai/sessions/:id/execute` | `POST` | 重新执行某条会话的 SQL 并返回最新数据 | 当前登录用户 |

> **提示**：为了节省前端渲染逻辑和保持数据实时性，复用时前端只需传入 `session_id`，后端根据 ID 查出记录的 SQL 并在当前项目中直接执行 `ExecuteQuery`，返回带数据的完整结构。

## 4. 前端交互设计

### 4.1 结果卡片追加「保存」操作
在现有的 AI 聊天对话流中，当 AI 成功返回了包含数据和图表的消息卡片 (`ChatMessage.vue`) 时，在卡片的右下角或右上角提供一个操作区：
- `[ 保存为有效分析 ]` (或者一个 ⭐️ 收藏图标)
- 用户点击后调用 `POST /api/v1/analytics/ai/sessions`，成功后按钮变为已保存状态。

### 4.2 头部增加「历史会话」入口
在 `/analytics/ai/index.vue` 顶部区域（如模型选择器旁边），添加一个按钮：
- `[ 历史分析记录 ]`
- 点击后触发 Element Plus 的 `el-drawer` 从右侧滑出。

### 4.3 抽屉页内容 (Drawer)
- 抽屉内展示一个列表 (可以是时间轴样式或卡片列表)。
- 每个列表项展示：用户的提问文本、生成时间、图表类型图标。
- 操作项：
  - **点击加载**：调用后端 `/execute` 接口，将结果作为一条新的 AI 消息追加到当前聊天窗口中，并附带标识“已加载历史分析”。
  - **删除**：移除该条记录。

## 5. 开发步骤拆解

1. **DB 与 模型层 (Backend)**：
   - 编写 `docs/sql/dmp_center.sql` 追加 `ai_chat_sessions` 表。
   - 编写 `internal/model/ai_chat_session.go` 实体类。
   - 编写 `internal/service/analytics/ai_session_service.go` 处理增删查与重新执行逻辑。

2. **API 路由层 (Backend)**：
   - 在 `internal/api/ai_chat.go` (或新建 handler) 编写并注册相关 API。

3. **前端 API 层 (Frontend)**：
   - 在 `src/api/analytics.ts` 中新增对应接口调用。

4. **前端组件改造 (Frontend)**：
   - 改造 `ChatMessage.vue`，增加收藏/保存交互。
   - 在 `index.vue` 中引入 `el-drawer` 编写历史列表组件。
   - 实现点击历史记录重新拉取数据并渲染图表的逻辑。
