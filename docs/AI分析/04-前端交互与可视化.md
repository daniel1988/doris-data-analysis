# 前端交互与可视化方案

当得到了安全且符合意图的 `SQL` 和在后台查出的二维结果集时，我们应该在前端直接进行可视化。体验参考业界优秀的 AI BI 方案（如 Metabase AI 或 Databricks 自带的交互查询）。

## 1. 菜单入口与交互形态布局 (Menu & UI Layout)

### 1.1 菜单入口定位
在系统现有的菜单导航体系中，将“AI 智能分析”（或“对话式洞察”）**直接挂载于左侧/顶部「分析模块」的菜单下**，与已有的事件分析、漏斗分析平级。
- 设定路由为 `/analytics/ai`。
- 使得用户在查阅常规分析报表时，能无缝、快速地切入到自由对话探索的数据模式中。

### 1.2 页面版式设计
在切入 `/analytics/ai` 路由后，建议采用类似当前主流的 Chat 左右布局或宽屏流式面板：
- **对话面板 (Left / Main Stream)**：包含用户发出的 `Message` 和助手回复的富文本流（Streaming Text）。
- **SQL 显示区**：LLM 成功输出后，会向前端吐出一个携带代码高亮的 SQL 模块，通过 `v-md-editor` 渲染，允许有经验的用户通过面板二次基于生成的 SQL 手动编辑和重新 run。
- **视图看板 (Dashboard Widget)**：在后端将该 SQL 发送给 Doris 执行结束后，将列结构 (Columns) 和横排数据 (Rows) 返回前端流中并被 Echarts 或表格包裹渲染。

## 2. 动图识别与可视化推荐 (Smart Viz)
LLM 除了输出 SQL，我们可以利用它**强大的结构化解析能力**去推荐可视化图样。

*通过 Function Calling 或 JSON 约定，要求 AI 除了 SQL 再输出一串渲染指令：*
```json
{
  "sql": "SELECT u_channel, COUNT(1) as cnt FROM user_data GROUP BY u_channel",
  "viz_type": "pie",
  "x_axis": "u_channel",
  "y_axis": "cnt",
  "narrative": "从结果看，大部分注册用户来源于 ios_appstore 渠道。"
}
```

### 2.1 ECharts 动态包裹
前端拿到这段 JSON 数据后，使用 `echarts`：
1. `viz_type === 'pie'` => 使用饼图；如果返回的是按天聚合的 `viz_type === 'line'` => 采用折线图。
2. 将 `columns` 中的 `x_axis` 映射为 `series.data.name` / `xAxis.data`。
3. 自动挂载 Tooltip 和 图例配置。

如果图表渲染错误，应当提供一个基础 fallback：始终展示一个带有分页的 `ElTable`，确保业务永远能获取出二维数据平面。

## 3. 保存入表与闭环
当用户认为这个 AI 分析产生的数据看板非常有效时，在生成的结果卡片右上角留出一个**【保存为看板 (Save to Dashboard)】** 按钮。
- 将上述的 `SQL + 图型参数` 视作为一个特殊的 Custom Query 看板项。
- 加入原有的 Dashboard 中，打通了“从任意想象力” => “标准化数据看板”的最短路径。
