# 04-数据结构与接口契约

## 1. 接口定义 (API)

### 1.1 执行分布分析
- **Endpoint**: `POST /v1/analytics/scatter`
- **Request**: `ScatterAnalysisReq`
- **Response**: `QueryResponse`

## 2. 前端类型定义 (TypeScript)

### 2.1 请求参数结构 (ScatterAnalysisReq)

```typescript
export interface ScatterAnalysisReq extends QueryRequest {
    /**
     * 分析指标
     */
    metric: EventMetric
    
    /**
     * 全局过滤条件
     */
    global_filter_groups: GlobalFilterGroups
    
    /**
     * 分组维度
     */
    groups: Group[]
    
    /**
     * 时间粒度（主要用于过滤 event_data 表）
     */
    time_grain: TimeGrain

    /**
     * 分布类型
     * 1: 离散值, 2: 自动分桶, 3: 自定义区间
     */
    scatter_type?: number

    /**
     * 自定义区间数组 (仅在 scatter_type=3 时有效)
     */
    scatter_ranges?: Array<{ min: number; max: number }>
}
```

### 2.2 响应数据结构 (QueryResponse)

```typescript
export interface QueryResponse {
    /**
     * 数据行，每行包含分组值、区间、用户数
     */
    rows: any[]
    
    /**
     * 列名数组
     */
    columns: string[]
    
    /**
     * 生成的 SQL 语句（用于预览）
     */
    sql: string
    
    /**
     * 查询耗时 (ms)
     */
    duration: number
}
```

## 3. 报表持久化 (Report)

报表存储在 `report` 表中，其 `query_params` 字段存储 JSON 序列化后的 `ScatterAnalysisReq`（移除 project_alias 等动态字段）。
`category` 字段固定为 `"scatter"`。
