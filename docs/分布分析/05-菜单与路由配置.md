# 05-菜单与路由配置

分布分析作为分析模块的核心功能之一，需要集成到系统的导航体系中。

## 1. 路由配置

在 `src/router/index.ts` 中新增分布分析的路由定义，确保其归属于分析模块。

### 1.1 路由定义示例

```typescript
{
  path: '/analytics/scatter',
  name: 'ScatterAnalysis',
  component: () => import('@/views/analytics/ScatterAnalysis.vue'),
  meta: { 
    title: '分布分析',
    icon: 'TrendCharts' // 使用 Element Plus 图标
  }
}
```

## 2. 侧边栏菜单

在 `src/layout/SidebarLayout.vue` 的 `subMenuItems` 计算属性中，将分布分析添加到分析模块的子菜单列表中。

### 2.1 菜单项配置

```typescript
// 在 analytics 模块的判断分支中添加
{ 
  path: '/analytics/scatter', 
  title: '分布分析', 
  icon: 'TrendCharts' 
}
```

## 3. 导航逻辑

- **Tab 标签页**：当用户进入分布分析页面时，`SidebarLayout` 会自动捕捉路由变化并在顶部的 `tags-view` 中新增“分布分析”标签。
- **项目关联**：分布分析属于项目维度功能，进入页面前需确保已在全局导航栏选择了有效的项目（`activeProjectAlias`）。
- **权限控制**：如果系统启用了按钮/菜单级权限，需在路由配置或后端菜单管理中关联对应的权限标识（如 `analytics:scatter:view`）。

## 4. 报表跳转支持

为了支持从“我的报表”或“看板”直接跳转回分布分析页面，路由需支持携带 `reportId` 参数：
- URL 格式：`/analytics/scatter?reportId=123`
- 页面初始化时（`onMounted`）应解析该参数并调用报表加载逻辑。
