# 01-功能概述与核心逻辑

分布分析（Scatter Analysis）用于分析用户在特定行为或属性上的数值分布情况。例如：
- 分析“付费”事件中，订单金额的分布情况（0-10元有多少人，10-50元有多少人等）。
- 分析“登录”事件中，用户等级的分布。
- 分析特定时间段内，用户触发某个事件的次数分布。

## 1. 核心逻辑

分布分析的本质是：
1. **筛选用户行为**：基于特定的事件（如支付、登录）和过滤条件。
2. **计算个体指标**：为每个用户计算一个数值（如总金额、平均值、最大值或简单的触发次数）。
3. **分桶（Binning）统计**：将计算出的个体指标值划分为不同的区间（桶），并统计落入每个区间的用户数量。

## 2. 关键概念

### 2.1 分析指标 (Metric)
用户配置的一个事件指标，系统会针对每个用户计算该指标的值。
- **事件**：如 `sys.order_pay`。
- **度量值**：如属性 `price` 的 `Sum`（求和），或者简单的 `Count`（次数）。

### 2.2 分布方式 (Scatter Type)
决定了如何划分区间：
- **离散值 (Discrete Value)**：不进行分桶，直接按具体数值展示。
- **默认区间 (Default Range)**：由系统自动计算合适的区间跨度。
- **自定义区间 (Custom Range)**：用户手动指定区间范围（如 0-10, 10-50, 50-100, 100+）。
- **百分位数 (Percentile)**：展示 10%, 20%...90% 分位数。

### 2.3 分组维度 (Groups)
支持按用户属性或事件属性进行分组，从而对比不同人群的分布差异。

## 3. 数据处理流程

1. **配置阶段**：用户选择一个指标、全局筛选条件、时间范围和分组维度。
2. **请求阶段**：前端构建 `ScatterAnalysisReq` 负载发送给后端。
3. **计算阶段**：
   - 后端首先生成 SQL，计算出 `(openid, group_fields..., metric_value)` 的中间结果集。
   - 然后对外层进行聚合统计，按 `group_fields` 和 `metric_value`（或其所属区间）统计 `COUNT(DISTINCT openid)`。
4. **展示阶段**：
   - 前端接收到结果（通常是二维数组），将其渲染为**柱状图（Bar Chart）**或**数据表格（Table）**。
   - 柱状图横轴为区间，纵轴为人数/用户占比。
