# 02-架构层重构：分析上下文与元数据管理

本计划旨在引入 `Provide/Inject` 模式，解决 Prop Drilling 问题，并集中化元数据（事件、属性、标签）的管理逻辑。

---

## 1. 目标
- 建立 `AnalysisContext`，简化跨层级组件通信。
- 集中管理项目元数据，减少重复请求与逻辑碎片化。
- 提供统一的状态刷新入口。

## 2. 执行步骤

### 步骤 1: 创建 AnalysisContext 定义
在 `src/views/analytics/context.ts` 中定义上下文结构：
- `state`: 包含 `projectAlias`, `metadata` (事件/属性列表)。
- `actions`: 包含 `refreshMetadata`, `triggerAnalyze` 等。

### 步骤 2: 在顶层组件 Provide 上下文
在 `EventAnalysis.vue` 中：
- 使用 `provide` 暴露 `AnalysisContext`。
- 将原本分散在顶层的 `projectAlias` 和从 `useEventMeta` 获取的数据放入上下文。

### 步骤 3: 子组件 Inject 上下文
重构以下组件，移除 `Props` 中的元数据传递，改为从上下文 `inject`：
- `MetricsList.vue` / `MetricItem.vue`
- `MetricPropertyPicker.vue`
- `BaseFilterItem.vue`
- `GlobalFiltersCard.vue`

### 步骤 4: 优化元数据加载逻辑
- 在上下文初始化时按需加载元数据。
- 增加元数据缓存机制，避免频繁切换项目时的闪烁或多余请求。

## 3. 验证标准
- [ ] 核心组件（如 `MetricItem`）的 `Props` 定义大幅减少。
- [ ] 切换项目后，上下文中的元数据能正确同步更新。
- [ ] 子组件能够直接调用上下文中的 `actions` 触发全局逻辑。
