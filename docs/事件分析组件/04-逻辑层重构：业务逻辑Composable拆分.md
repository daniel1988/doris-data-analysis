# 04-逻辑层重构：业务逻辑 Composable 拆分

本计划聚焦于将巨石 Composable `useEventAnalysisForm` 拆解为职责单一的小型 Hook，提升代码的可维护性与测试性。

---

## 1. 目标
- 职责分离：指标、过滤、分组逻辑互不干扰。
- 逻辑复用：拆分出的 Hook 可被漏斗分析、留存分析快速复用。
- 简化状态同步逻辑。

## 2. 执行步骤

### 步骤 1: 拆分指标状态管理 (`useMetricState`)
- 负责 `event_metrics` 的增删改查。
- 处理指标重命名的联动逻辑。
- 处理自定义公式指标的 Token 转换。

### 步骤 2: 拆分过滤与分组状态 (`useFilterState` / `useGroupState`)
- 管理 `filter_groups` 和 `groups`。
- 提供“一键清空”、“预设模板加载”等功能。

### 3. 重构执行器 (`useAnalysisRunner`)
- 整合上述状态，构建最终的 `Payload`。
- 处理 API 调用细节、请求超时、错误拦截。
- 将结果解包逻辑独立，支持不同格式的后端响应。

### 4. 状态聚合
在 `useEventAnalysisForm` 中引用上述 Hook，保持对外的 API 兼容性，但内部逻辑已完成解耦。

## 3. 验证标准
- [ ] 代码逻辑清晰，单个 Hook 文件长度显著减少。
- [ ] 单元测试可以针对特定的业务逻辑（如指标增删）独立运行。
- [ ] 页面功能无退化，分析请求依然能够正确发送并接收。
