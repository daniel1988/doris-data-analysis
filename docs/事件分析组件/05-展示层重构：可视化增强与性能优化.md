# 05-展示层重构：可视化增强与性能优化

本计划通过解耦展示逻辑与优化底层渲染技术，显著提升分析结果的展示效果与系统响应速度。

---

## 1. 目标
- 彻底解决大数据量下 ECharts 的卡顿问题。
- 提供更专业的图表交互与数据导出能力。
- 增强表格的可用性（分页、多列排序）。

## 2. 执行步骤

### 步骤 1: ResultView 逻辑解耦
将 `ResultView.vue` 拆分为：
- `ChartView.vue`: 专注图表渲染、配色方案、类型切换。
- `TableView.vue`: 专注数据明细、分页、格式化、排序。

### 步骤 2: ECharts 性能深度优化
- **shallowRef**: 使用 `shallowRef` 存储 ECharts 实例，防止 Vue 对 ECharts 内部数据进行深度响应式追踪。
- **setOption 优化**: 使用 `notMerge: true` 模式，确保数据更新时图表平滑过渡而非闪烁重绘。
- **Resize 优化**: 引入 `ResizeObserver` (通过 VueUse 的 `useResizeObserver`)，精准触发图表重绘。

### 步骤 3: 表格功能增强
- **分页机制**: 实现真正的分页（前端分页或后端分页配合），避免一次性渲染数千行 DOM。
- **列管理**: 支持用户手动显隐特定列。
- **格式化**: 针对数值列自动应用千分位、百分比等格式化处理。

### 步骤 4: 导出功能落地
- 增加“导出 Excel”按钮，利用 `xlsx` 库将当前表格数据本地化。
- 增加“图表另存为图片”功能。

## 3. 验证标准
- [ ] 页面在切换图表类型时无明显白屏或延迟。
- [ ] 调整浏览器窗口大小时，图表能自动填满容器。
- [ ] 导出文件内容准确，格式符合预期。
