# 03-原子层重构：属性选择器与通用过滤器

本计划通过重构最底层的交互原子（属性选择器、过滤器项），提升系统的组件复用率与交互一致性。

---

## 1. 目标
- 抽象通用的 `UnifiedPropertySelector`，统一处理属性与标签选择。
- 增强 `BaseFilterItem` 的数据类型感知能力。
- 提升底层 UI 组件的性能与交互流畅度。

## 2. 执行步骤

### 步骤 1: 构建 UnifiedPropertySelector
创建 `src/views/analytics/components/shared/UnifiedPropertySelector.vue`：
- **功能**: 支持 Tab 切换（事件属性、用户属性、标签、分群）。
- **交互**: 统一的搜索过滤、类型图标展示、虚拟列表支持。
- **重构**: 替换 `PropertySelect.vue` 和 `MetricPropertyPicker.vue` 中的冗余代码。

### 步骤 2: 增强 BaseFilterItem
- **类型识别**: 根据所选字段的 `dataType` 自动切换操作符列表（如：数值显示“大于”，布尔显示“为真/为假”）。
- **动态值编辑器**: 
    - String -> 多选下拉或文本框。
    - Number -> 数字范围输入。
    - Date -> 动态/静态日期选择器。

### 步骤 3: 优化过滤器弹窗交互
- 优化 `el-popover` 或 `el-dialog` 的挂载时机，实现懒加载。
- 增加“非法值”校验提示。

## 3. 验证标准
- [ ] 属性选择器在不同场景下的外观与交互完全统一。
- [ ] 过滤器能够根据字段类型准确限制操作符。
- [ ] 属性列表超过 100 条时，滚动与搜索无明显卡顿。
