# 04-日志明细全链路流程

## 1. 查询执行链路 (Execution Flow)

### 1.1 前端构造请求 (Request Phase)
- 用户点击“查询”或变更分页。
- `loadData` 函数从 `searchForm` 和 `activeFilters` 获取当前所有搜索和过滤条件。
- 构造 `EventDetailRequest` Payload：
  - 合并 `project_alias`、`page_size`、`page_num`。
  - 构造 `select_fields`（如 `e_event_time`, `e_openid`, `e_event_id`, `e_ip`, `e_properties`）。
  - 将快速搜索（如 `openId` 等）与高级过滤条件合并到 `event_filter_group.filters` 数组中。

### 1.2 发送请求 (Transport Phase)
- 调用 `api/eventDetail.ts` 中的 `getEventDetail(params)`。
- 通过 Axios 实例发送 `POST` 请求。

### 1.3 后端处理 (Backend Processing Phase)
- 后端根据 `project_alias` 路由到对应的 Doris 数据库实例。
- 解析 `event_filter_group` 中的过滤条件。
- 动态生成 SQL 查询语句（基于 `select_fields` 和 `filters`）。
- 执行 SQL 查询并获取 Doris 结果集。
- 将结果转换为 `EventDetailItem` 数组并返回给前端。

### 1.4 前端渲染 (Rendering Phase)
- 前端接收到 Response。
- `loadData` 更新 `tableData` 响应式引用。
- 提取 Response 中的 `data` 列表及 `count` 总数，更新 `pagination.total`。
- `el-table` 根据 `tableData` 自动渲染表格行。
- 如果存在 `e_properties`，渲染“详情”按钮，供用户点击查看 JSON 详情。

## 2. 交互逻辑细节

### 2.1 过滤器变更 (Filter Updates)
- 当用户在快速搜索栏或高级过滤器弹窗中修改条件。
- 触发 `saveFiltersToStorage`。
- 调用 `loadData` 重新发起请求。
- 实现“即搜即得”的交互体验。

### 2.2 项目切换 (Project Switch)
- 监听 `appStore.activeProjectAlias` 变化。
- 当项目别名变更时：
  - 尝试从 `localStorage` 加载新项目的过滤器。
  - 清空旧数据。
  - 自动执行新项目的首次查询。

### 2.3 异常处理
- 如果接口返回 `code !== 0`，通过 `ElMessage.error` 提示用户。
- 如果 Response 为空或结构异常，前端通过防御性编程（如 `|| []`）避免页面崩溃。
- 数据加载期间显示 `v-loading` 遮罩。
