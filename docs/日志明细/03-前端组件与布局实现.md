# 03-前端组件与布局实现

## 1. 页面布局与交互流程

### 1.1 菜单入口与路由
- **模块归属**：位于“元数据 (Metadata)”主模块下。
- **菜单名称**：日志明细。
- **路由路径**：`/metadata/log`。
- **视觉标识**：使用 `Memo` 图标。

### 1.2 页面布局 (LogDetail.vue)
- **Header**: 包含页面标题、说明文字。
- **Search Section**:
  - **Quick Search Form**: `openId` (用户ID)、`eventId` (事件ID)、`eventTimeRange` (时间段)。
  - **Filter Controls**: 提供“查询”和“过滤器”按钮。
  - **Active Filters Bar**: 展示已应用的高级过滤器卡片，支持直接在卡片上修改值。
- **Table Section**:
  - **Data Table**: 响应式表格，支持列锁定、排序和自定义列内容。
  - **Pagination**: 底部居中显示，支持自定义每页条数。

## 2. 核心逻辑实现 (useEventDetail.ts)

#### 1.2.1 状态管理
- `loading`: 数据加载状态。
- `tableData`: 当前页显示的事件数据。
- `activeFilters`: 已应用的高级过滤器列表。
- `searchForm`: 快速搜索表单数据。
- `pagination`: 分页器状态。

#### 1.2.2 搜索与过滤逻辑
- **快速查询**: 用户点击“查询”按钮时，触发 `handleSearch` 函数，重置分页并执行 `loadData`。
- **高级过滤**:
  - 用户点击“过滤器”按钮，弹出 `EventFilterDialog`。
  - 用户在弹窗中选择字段、操作符并输入值，保存后更新 `activeFilters`。
  - `loadData` 会自动合并 `searchForm` 和 `activeFilters` 为最终的 `FilterGroup`。
- **自动保存**: 每次过滤器变更后，调用 `saveFiltersToStorage` 将状态持久化到 `localStorage`。
- **初始化加载**: `onMounted` 阶段，优先从 `localStorage` 加载保存的过滤器，随后自动执行首次查询。

### 1.3 关键 UI 特色 (UI/UX Details)

#### 1.3.1 属性详情查看 (EventPropertiesDrawer.vue)
- 当用户点击“详情”按钮时，弹出抽屉组件。
- `e_properties` JSON 字符串会通过 `JSON.parse` 解析，并使用缩进进行美化展示。
- 方便开发和运营人员快速校验事件携带的具体业务参数。

#### 1.3.2 快捷操作
- **复制字段**: 通过监听 `dblclick` 事件，结合 `navigator.clipboard.writeText` 实现双击快速复制 `openId` 等长字符串字段。
- **状态标记**:
  - `platform`: 根据平台名称显示不同的标签颜色（如 Android 绿色，iOS 橙色）。
  - `eventId`: 使用 `el-tag` 包裹，提升识别度。

#### 1.3.3 过滤器交互增强 (ActiveFiltersBar.vue)
- 在搜索框下方，实时显示已激活的高级过滤条件。
- 采用紧凑型设计，支持在不打开弹窗的情况下，直接通过 `DimensionValueSelect` 修改过滤值。
- 支持快速移除单个过滤条件。
