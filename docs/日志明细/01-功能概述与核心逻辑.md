# 01-功能概述与核心逻辑

## 1. 目标
实现一个高效、易用的日志明细（事件明细）查询系统，允许用户实时查看 Doris 数据库中记录的原始事件数据，并支持灵活的过滤和检索。

## 2. 核心功能
- **实时查询**：支持按项目别名查询最新的事件明细。
- **快速检索**：支持按用户 ID (`e_openid`)、事件 ID (`e_event_id`)、事件名称 (`e_event_name`)、应用包名 (`e_package_name`) 等进行快速过滤。
- **高级过滤**：支持多维度、多操作符（等于、包含、在范围内、不等于、为空等）的复杂条件组合。
- **自定义属性展示**：支持以 JSON 格式展开查看事件携带的 `e_properties` 自定义属性。
- **便捷操作**：支持双击快速复制关键字段（如用户 ID、IP 地址、请求 ID 等）。
- **状态持久化**：自动保存用户的查询习惯和过滤器设置，提升重复查询的效率。

## 3. 技术栈
- **前端框架**：Vue 3 (Composition API)
- **UI 组件库**：Element Plus
- **API 通信**：Axios (基于统一封装的 request 工具)
- **状态管理**：Reactive / Ref (Composition API)
